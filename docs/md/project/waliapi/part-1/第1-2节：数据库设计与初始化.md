## 二、技术选型说明

### 2.1 为什么是 SQLite？

WaLiAPI 是一个本地桌面应用，数据库的选择有几个候选：

| 方案 | 优点 | 缺点 |
|---|---|---|
| SQLite | 零配置、单文件、随应用分发、性能好 | 不适合多进程并发写 |
| JSON 文件 | 简单 | 无索引、查询困难、无事务 |
| 嵌入式 RocksDB | 性能极高 | 编译慢、二进制体积大 |

对于桌面应用来说，**SQLite 是最优解**：零配置、单文件存储在应用数据目录、支持 SQL 查询和索引、有成熟的迁移机制。

### 2.2 为什么是 sqlx？

Rust 生态中操作 SQLite 的库主要有两个：
- **rusqlite**：同步 API，简单直接，但在 async 环境中会阻塞线程
- **sqlx**：异步 API，编译时检查 SQL，支持连接池，与 Tokio 深度集成

我们使用 `sqlx` + `runtime-tokio`，配合 `SqlitePoolOptions` 建立连接池。
